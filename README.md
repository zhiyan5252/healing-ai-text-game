# Witness — 一个见证型叙事互动原型

这是一个基于 **Streamlit** 构建的文字互动原型，  
探索一种不同于“安慰 / 建议 / 解决问题”的人机互动方式。

在这个项目中，AI（或叙述者）不提供答案，  
而是作为一种**见证型存在（Witness-type Presence）**，  
陪伴用户直面自身处境，并允许他们自行决定是否继续。

---

## 项目背景

在大量情绪类产品与对话式 AI 中，常见的设计倾向是：

- 快速安慰
- 给出建议
- 引导积极情绪
- 尝试“解决问题”

但在真实体验中，这类回应有时会带来新的压力：
- 被要求“振作”
- 被暗示“应该更积极”
- 被过早推向行动，而尚未准备好

本项目尝试探索一种**更克制的互动方式**：
> 不判断、不说教、不替代选择，只见证。

---

## 核心理念：见证型存在（Witness-type Presence）

在这个原型中，叙述者遵循以下原则：

- 不对用户情绪做价值判断
- 不提供直接建议或解决方案
- 不强行引导“积极转变”
- 明确自身能力与边界

叙述的重点不是“改变用户”，  
而是**为用户提供一个可以停下来、被看见的空间**。

---

## 当前实现内容（第一幕）

目前项目实现了一个**完整可运行的第一幕体验流程**：

1. **开屏引导**  
   提供一个低压力的进入场景，强调“不着急”。

2. **情绪放置区**  
   用户可以自由输入当前最明显的感受，不要求整理或解释。

3. **情绪停顿区**  
   暂停推进节奏，允许用户选择是否继续。

4. **直面表达区**  
   邀请用户描述一个近期反复出现的困扰（可模糊、不完整）。

5. **见证式回应**  
   提供克制、不替代判断的文本回应。

6. **托底结束**  
   明确结束边界，允许用户离开或之后再回来。

整个流程基于 **状态机（step-based flow）** 实现，  
强调结构清晰、可扩展、可复用。

---

## 技术实现

- Python 3.x
- Streamlit
- 状态机驱动的页面流程（session_state + step）
- 模块化的跳转与重置逻辑

当前版本**未接入大模型 API**，  
所有文本均为手工设计，用于专注打磨体验与结构。

---

## 为什么暂不接入 AI 模型？

在体验结构与边界尚未稳定之前，  
过早引入生成式模型可能会掩盖以下问题：

- 叙事位置是否清晰
- 是否存在隐性说教
- 是否越过用户边界
- 是否在“替用户做决定”

因此，本项目选择先完成**结构与理念的验证**，  
再考虑 AI 能力的引入方式。
